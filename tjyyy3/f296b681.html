2. 项目介绍

  • 技术栈：Springboot、mybatis、maven、redis、jwt、Lombok、validation、pageHelper

  • 基本流程：

    1. XxxxController.java：
      • 控制器层通过注解（如 @GetMapping、@PostMapping 等）定义对应的接口映射。
      • 控制器接收 HTTP 请求，提取参数（如路径变量、请求体等），然后调用 XxxxService 中定义的业务方法。
    2. XxxxService.java：
      • Service 层通常定义业务接口，声明业务方法。
      • Service 层负责处理请求中的业务逻辑，但它本身不直接实现这些业务逻辑，而是委托给具体的业务实现类（通常是 XxxxServiceImpl.java）来执行。
    3. XxxxServiceImpl.java：
      • 这是 Service 层的具体实现类，真正执行业务逻辑的地方。
      • 在这个类里，可以调用数据访问层（如 XxxxMapper）来进行数据库操作，处理增、删、改、查等功能。
    4. XxxxMapper.java：
      • Mapper 层是数据访问层，直接与数据库交互。
      • 使用 MyBatis（Mapper）来处理与 MySQL 数据库的增、删、改、查操作。

3. 项目功能

  1. 用户：注册、登陆、修改密码、修改头像、获取用户详细信息、更新用户基本信息

    • 注册：用户名和密码需要进行validation（用户名是否已存在、长度…），保存信息至MySQL（对密码要加密）
    • 登陆功能：基于用户名和密码，验证成功后生成jwt令牌，并保存至redis
    • 修改密码功能：更新数据库中的密码，更新jwt令牌，在redis里删除旧jwt token，使旧token失效
    • 其他功能：执行前先比对token是否与redis里的token一致且没有过期
    • ps：jwt和存储至redis里的token都最好设置过期时间，否则数据会越来越多（造成内存浪费）

  2. 文章分类：新增文章分类、更新文章分类、获取文章分类详情、删除文章分类、文章分类列表

    • 增删改查
    • 文章分类列表：PageHelper实现分页查询（可以设置的参数：每页的条数pageNum，查询第几页pageSize）

  3. 文章管理：发布文章、更新文章、获取文章详情、删除文章、文章列表（条件分页）

    • 增删改查
    • 文章列表（条件分页）：因为有两个属性不是必须提供，所以可以使用动态SQL进行查询

  4. 其他：文件上传

    • 使用阿里OSS
