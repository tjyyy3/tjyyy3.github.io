题目描述

求把 N×M 的棋盘分割成若干个 1×2 的长方形，有多少种方案。

例如当 N=2，M=4 时，共有 5 种方案。当 N=2，M=3 时，共有 3 种方案。

输入格式

输入包含多组测试用例。

每组测试用例占一行，包含两个整数 N 和 M。

当输入用例 N=0，M=0 时，表示输入终止，且该用例无需处理。

输出格式

每个测试用例输出一个结果，每个结果占一行。

分析

![dp分析图](../../../../../personal doc/OneDrive/个人笔记/002 附件/Pasted image 20240501213357-1716039774724.png)

  • 前提：方块可以横着摆，也可以竖着摆，但是只要横着放的确定了，那么竖着放就也确定了
  • 对于第 i 列来说，如果有第 i-1 列的方块伸过来则记为 1，否则为 0。
  • 则可以用二进制数来表示第 i 列的状态。如：第 3 列为 11001，第 2 列为 00100
  • 若要构成一个可行的方案，必须满足两个条件：
    1. 相邻两行的方块不能冲突
      -> 第 i 列的状态 & 第 i-1 列的状态 == 0 （如果&完是 1 说明有两个方块冲突了）
    2. 不能有连续奇数个空着的位置（不能有连续奇数个 0）
  • 使用 f[i, j]来记录 0第 i 列的总方案数。j 是 02^n-1，记录第 i 行可能的状态
  • 最后只需要使用一个循环，遍历第 i 行和第 i-1 行所有可能的状态的组合
  • F[m][0]就是答案（第 m 列没有任何一个方块伸出来，说明方块充满了 0~m-1 列）

参考代码

注意点：

  • 要注意运算优先级。进行 1<<n 等类似移位操作时最好加上 ()，否则像 j < (1 << n) 等代码不一定会按预期的运算从而导致错误
  • 每一轮都要重置 f （memset），因为每一次的 m 和 n 不一样
  • 进行预处理的时候（用 st[] 记录不包括连续个奇数 0 的可能状态时）要先将 st[i] 置为 true，或者在循环外先对 st 进行 memset。
  • 进行预处理的最后要记得最后一次对 cnt 进行判断
