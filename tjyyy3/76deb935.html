【前端】学习笔记1

一、CSS

1. 盒子 box

  1. 盒子的组成：

    内容 content + 内边距 padding + 边框 border + 外边距 margin

  2. 盒子的类型（使用box-sizing设置）

    1. 内容盒子 content-box （Default）

      内容 content + 内边距 padding + 边框 border + 外边距 margin

      • 特点：盒子的宽 = width + padding + border ； 盒子的高 = height + padding + border

    2. 边框盒子 border-box (又称怪异模型)

      内容 content ( 包含【内边距 padding + 边框 border】) + 外边距 margin

      • 特点：响应式布局，css中width和height设置的是盒子的最终的宽高

    3. 特殊的盒子：替换元素

      常见的替换元素有<img> 、 <iframe> 、 <video> 等
      替换元素加载的是外部内容，并不会受现有的 CSS 属性的影响。

2. CSS选择器的优先级

前情提要：CSS的特性：继承性、层叠性、优先级

7. CSS可以继承的属性

子元素可以继承父类元素的样式

  1. 字体的一些属性：font（包括大小、颜色等）
  2. 文本的一些属性：line-height，text-align
  3. 元素的可见性：visibility：hidden
  4. 表格布局的属性：border-spacing
  5. 列表的属性：list-style
  6. 页面样式属性：page
  7. 声音样式的属性
  8. …(不常见)

8. 预处理器（SASS、 LESS）

预处理器增加了变量、函数、混入等强大的功能

二、JavaScript

1. JS由哪三部分组成？

  1. ECMAScript：JS的核心部分，定义了语言的 基础语法和功能，如变量声明（var、let、const）、流程控制（for、if）、数据类型（数组、字符串、对象等）。
  2. 文档对象模型（DOM）：DOM把整个HTML页面规划为元素构成的文档
  3. 浏览器对象模型（BOM）：对浏览器窗口进行访问和操作，包括操作窗口、导航、浏览器历史、定时器等功能。

2. JS有哪些内置对象

String、Boolean、Number、Array、Object、Function、Math、Date、RegExp…

3. 操作数组的方法有哪些？

concat(),join(),push(),pop(),sort(),splice(),unshift(),shift(),reverse(),map(),filter(),some(),reduce(),isArray(),findIndex(),every()…

3.1 常见函数解释

  1. join()：将数组中每个元素使用分割符（默认是’,’）连接成一个字符串

  2. splice()：添加、删除或替换数组元素
    array.splice(start, deleteCount, item1, item2, ...)
    start是修改开始的位置，deleteCount是说要删除几个元素，item1, item2可选
    deleteCount=0时，后面的item1，item2等是要添加的元素

  3. unshift()：在数组开头添加1个或者多个元素，返回新的数组长度

  4. shift()：移除并返回数组的第一个元素

  5. map()：对数组中每个元素执行回调函数，返回新数组

  6. some():测试数组中是否至少有一个元素通过回调函数，返回值是boolean

  7. every()：测试数组中是否所有元素都通过回调函数，返回值是boolean

  8. reduce()：对数组中的每一个元素执行回调函数，最终将结果汇总成一个单一的值。可以用来进行累加、累乘、扁平化数组、计算最大值/最小值等操作。

  9. isArray()：判断给定值是否为数组，返回值是boolean

  10. findIndex()：返回数组中第一个符合回调函数的元素的索引，如果没有，返回-1

3.1 哪些方法会改变原数组？

push(),pop(),unshift(),shift(),sort(),reverse()

4. JS对数据类的检测方式有哪些?

  1. typeof:能判断基础类型，不能判断引用类型（只会显示是object/除了函数，函数会显示function）
  2. instanceof:能判断引用类型，不能判断基础类型
  3. construct：返回函数的构造函数引用，基本能够判断基本类型和引用类型，但可能因为构造函数重写而不准确
  4. obj.prototype.toString.call():最全面的一个，都可以判断。返回格式为 "[object Type]"

5. 说一下闭包，闭包有什么特点?

  1. 闭包定义：在函数内部定义另一个函数，并且内部的函数可以访问外部函数中的变量（即使在外部函数执行结束之后）
  2. 特点：
    1. 数据持久化。闭包可以让函数里的变量执行结束后依旧保存在内存中，形成持久化的状态。
    2. 可能会造成内存泄露。因为1可能会导致内存无法被及时回收，造成内存泄露
    3. 可以用来封装私有变量。只暴露必要的接口，使得代码更加安全和模块化

6. 前端的内存泄漏怎么理解?

内存泄露是指不需要的内存没有被及时释放，导致内存一直被占用，最后导致性能下降甚至崩溃。

内存管理主要依赖垃圾回收机制（Garbage Collection，GC）。GC会自动查找和释放不再使用的内存空间

7.事件委托是什么?

事件委托是将子元素的事件绑定到父元素上，通过事件冒泡的方式来捕获子元素的事件。

优点：可以减少事件处理器的数量，提升性能。适合动态内容和大量相似元素的事件处理

8.基本数据类型和引用数据类型的区别?

  1. 基本数据类型：String, Boolean, Number, Undefined, null, Symbol(ES6引入)
    • 变量存储的是值，值直接保存在栈中
  2. 引用数据类型：Object(Array, Date, Map(ES6引入), Set(ES6引入)), Function
    • 变量存储的是指向堆的地址（引用），值存储在堆中

9.说一下原型链

原型链是JS实现继承的基础，原型链可以让对象之间共享属性和方法。

每一个对象都有一个内部属性[[prototype]](__proto__)，指向它的原型对象；它的原型对象又会有一个内部属性[[prototype]](__proto__)，以此类推最后指向null

person1 → Person.prototype → Object.prototype → null

10.new操作符具体做了什么?

  1. 创建一个新的空对象：
  2. 将新对象的[[prototype]]指向构造函数的prototype对象:obj.__proto__ = Person.prototype;
    • 这一步使obj通过原型链继承构造函数的属性和方法
  3. 将构造函数内部的this绑定到新对象上:const result = Person.call(obj, 'Alice'); // this 指向 obj
    • 初始化obj的属性和方法
  4. 根据构造函数的返回值决定返回什么
    • 如果构造函数显式返回的是对象，new就会返回这个对象
    • 如果构造函数没有返回对象，则返回步骤1中创建的对象

11.JS是如何实现继承的?

  1. 通过原型链进行继承

    • 缺点：会导致所有实例共享父类的引用属性，如果一个实例修改了引用属性，其他实例也会受到影响

  2. 通过构造函数进行继承

    • 在子类的构造函数中调用父类的构造函数。通过call或者apply方法将this绑定到子类的实例上，继承父类的属性
    • 缺点：可以继承属性，且避免了引用属性共享的问题，但是无法继承父类的方法

  3. 组合继承（结合原型链和构造函数）（ES5常用方法）

    • 子类可以继承父类的属性和方法
    • 使用Object.create()创建新的原型对象，避免了引用属性共享的问题

  4. class继承（ES6）

    • 使用extends关键字实现继承
    • 必须使用super()执行父类的构造函数

12.JS的设计原理是什么?

  1. JS引擎
    • 内存分配、垃圾回收
    • 优化代码执行，提高效率
    • 将JS代码解析为机器代码
  2. 运行上下文：JS执行代码时的环境，用于管理变量、函数、对象的访问和执行，决定this指向哪个对象
    • 全局上下文
    • 函数上下文
    • 块级上下文（ES6）
  3. 调用栈：管理函数调用
    • 当一个函数被调用的时候会将其上下文推入（push）调用栈，当函数调用完毕后会从栈顶弹出
    • 当调用嵌套函数时，内层函数会被push进栈顶
    • 如果函数递归调用过多可能会导致栈溢出（Stack Overflow）
  4. 事件循环：实现异步处理的核心机制。管理异步操作（定时器、事件监听、网络请求等），确保其在正确的时间执行
    • JS执行代码时会先执行同步任务，将其push进调用栈
    • 异步任务会被放入任务队列（Task Queue）
    • 事件循环会不断检查调用栈是否为空，是的话就将任务队列里的任务push进调用栈执行
  5. 回调callback：异步编程的基础，(定时器、事件监听、网络请求等)操作完成后，会调用回调函数来处理结果

13.JS中关于this指向的问题

  1. 全局中的this：指向window
  2. 作为方法调用：指向调用该方法的对象
  3. 构造函数中的this：使用new调用构造函数时，指向新创建的实例对象
  4. 箭头函数中的this：从外部作用域（词法作用域）“继承”this的值
  5. 显式绑定this：
    1. call 和 apply ：将this绑定到第一个参数
    2. bind：返回一个新的函数，并将this永久绑定为指定对象，不会立刻调用
  6. DOM事件处理函数中的this：默认指向触发事件的DOM元素

14.script标签里的async和defer有什么区别?

都是用来控制外部JS脚本的加载和执行时机

  1. async：

    异步加载，不会阻塞HTML解析，脚本加载完后马上执行（可能在HTML解析完成前）
    async脚本的执行顺序不一定按照<script>标签的顺序，而是谁先加载完谁先执行

  2. defer:

    异步加载，不会阻塞HTML解析，但是要在HTML完全解析完成后，按照<script>标签的顺序依次执行

    适合依赖HTML结构的脚本，确保在HTML解析完成后按顺序执行

15.setTimeout最小执行时间是多少?

setTimeout的最小执行时间是4ms

setIntervals的最小执行时间是10ms

（即不满足最小执行时间会按最小时间算）

16.ES6和ES5有什么区别?

ES5：ECMAScript5, 2009年进行第五次修订，又称ECMAScript2009
ES6:ECMAScript6, 2015年进行第五次修订，又称ECMAScript2015，是JS的下一个版本标准

17.ES6的新特性有哪些?

  1. 新增块级作用域(let,const)
    • 不存在变量提升：比如var可以使变量在声明前被使用（但是值是Undefined）
    • 存在暂时性死区问题：在变量被声明前无法使用
    • 块级作用域的内容
    • 不能在同一个作用域里重复声明
  2. 新增了定义类的语法糖 (class)
  3. 新增了一种基本数据类型 (symbo1)
  4. 新增了解构赋值
  5. 新增了函数参数的默认值
  6. 给数组新增了API
  7. 对象和数组新增了扩展运算符
  8. Promise
    • 解决回调地狱的问题
    • 自身有all,reject,resolve,race方法
    • 原型上有catch,then
    • 把异步操作队列化
    • 有三种状态：pedding初始状态，fullfilled操作成功，rejected操作失败
    • 一旦发生，状态就会凝固，不会再改变。只有两个状态：pedding -> fullfilled 和 pedding -> rejected
  9. 新增了模块化 (import,export)
  10. 新增了set和map数据结构
  11. 新增了generator
  12. 新增了模板字符串
  13. 新增了箭头函数（大概率问你箭头函数和普通函数的区别）
    • 不能作为构造函数使用，不能用new => 箭头函数没有原型
    • 不能使用call,apply,bind改变this的指向
    • this指向外层第一个函数的this
    • 箭头函数没有arguments
