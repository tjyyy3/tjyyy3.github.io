hexo搭建博客初体验

为什么想搭建博客？

在度过几年的大学生活后，我发现自己频繁地遭遇着相同的问题，而解决方法总是不外乎上网搜索，浏览一系列 CSDN 文章，最终解决问题。然而，过了一两周，甚至一个月后，如果再次遇到相同的问题，我却又陷入了搜索、查看、修正、遗忘的循环。为了解决这个问题，我尝试将遇到的问题记录在本地电脑。然而每次保存的地方都不一样，找起来麻烦，甚至忘了自己曾经做过记录。因此，我突然冒出一个想法：为什么不自己搭建一个网站来记录自己曾经踩过的坑呢？而且，搭建的博客过程，也是我成长、学习知识的过程~

工具

  • Hexo
  • Github page
  • Github action
  • Butterfly（主题美化）
  • ……

参考教程

  1. 使用 Github Action 自动部署 | 安知鱼 (anheyu.com)
  2. 基于 Hexo 从零开始搭建个人博客系列 | 唐志远の博客 (fe32.top)

踩到的坑（注意点）

  1. 安装 butterfly 插件之后，为了减少升级主题后带来的不便需要复制一份 _config.yml文件到 Hexo 根目录中。此时注意这个 _config.yml 文件是 butterfly 中的文件，复制一份到根目录（本项目中为 blog-demo）
    ![路径](../../../../../personal doc/OneDrive/个人笔记/002 附件/Pasted image 20240420210110-1716039834808.png)
  2. 使用 github action 的时候，复制代码到 autodeploy.yml 时要注意其中的具体内容
  3. 如果自己 github 上仓库的分支是 main，代码里的 master 都要改成 main
  4. 如果没有增加 bilibili 番剧更新和博客更新提醒功能的话，hexo bangumi -u * 和 npm install gulp-cli -g 、gulp* 注意删除，不然很大可能在执行自动化的时候报错
  5. 在提交源码到仓库前一定要先把[Blogroot]/themes/butterfly/.git先删了，否则无法将 butterfly 文件夹上传上去
