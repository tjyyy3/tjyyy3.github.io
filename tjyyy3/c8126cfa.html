【vue2】day1项目实战

1. 今日进展

  1. 进行了环境配置及基本工具的安装

  2. 了解了一些基本概念

  3. 对网站模块进行了初步的划分

2. 今日收获

1. nvm

  • 一个用于管理和切换不同版本的 Node.js 的工具。

  • 能够在同一台机器上安装和使用多个版本的 Node.js，并能够在这些版本之间快速切换

ps：需要先卸载 node.js 之后再下载安装 nvm

2. vue的单根元素规则

每个 Vue 组件的模板（<template> 标签中的内容）必须有且只有一个根元素。也就是说，模板中所有其他的 HTML 元素都必须嵌套在这个唯一的根元素之下。

  • 为什么只能有一个根元素？
    1. 在Vue中每个组件都是一个单独的节点，如果有多个元根素，Vue无法区分哪个才是组件的主入口。会导致不明确的渲染行为
    2. Vue使用虚拟DOM来追踪和更新视图。一个组件对应一个虚拟DOM节点。
    3. 便于管理组件的状态和生命周期
  • Vue3支持多个根结点（“Fragments”（片段）），但还是更推荐使用单节点

3. vue-router 路由

Vue Router 允许你在不同的页面（视图）之间进行切换而不需要重新加载整个页面，从而提供更快、更流畅的用户体验。

  • 路由（Routing） 是指通过 URL 地址映射到不同的页面或视图的过程。
  • vue-router功能：
    • 动态路由匹配：Vue Router 支持使用参数来定义动态路径，以便在路径中捕获动态值（如用户 ID）。
    • 嵌套路由：支持在页面内部使用嵌套的视图结构，这允许你在一个父视图中显示子视图。
    • 懒加载路由：通过异步组件和 Vue 的 Webpack 配置，你可以按需加载路由组件，减少初始加载时间，提升性能。
    • 创建单页面应用（SPA）：通过 Vue Router，Vue 应用可以模拟多页面的行为，用户在不同“页面”之间切换时，实际上是加载不同的组件，而不是重新加载整个页面。

4. 组件

组件可以看作是可重用的代码块，它封装了特定的功能或 UI 部分，以便在多个地方使用。组件的主要目的是提高代码的可重用性、模块化和可维护性。

在 Vue 中，一个组件通常由以下几个部分组成：

  1. 模板（Template）：定义组件的 HTML 结构。

  2. 脚本（Script）：定义组件的逻辑和行为，如数据、方法、生命周期钩子等。

  3. 样式（Style）：定义组件的样式，可以是全局样式或局部样式。

3. 今日总结

  1. 进行项目的时候要注意各个依赖、包之间的版本关系，有时候跑不起来、有error是因为版本冲突/不兼容
  2. 要学会利用前人造好的轮子（框架），比如 vue-cli, element-ui等等
